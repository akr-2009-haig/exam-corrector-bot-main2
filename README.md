# Exam Corrector Bot — منصة الامتحانات

بوت تيليجرام + Mini App لتصحيح أوراق الإجابة تلقائياً باستخدام الذكاء الاصطناعي. يرفع الأستاذ مفتاح الإجابة، ثم يرسل الطلاب صور أوراقهم ليحصلوا على التصحيح فوراً مع النقاط والتصنيف.

---

## متغيرات البيئة المطلوبة

### البوت (Bot)
| المتغير | مطلوب | الوصف |
|---------|:---:|-------|
| `BOT_TOKEN` | ✅ | توكن بوت تيليجرام من @BotFather |
| `ADMIN_IDS` | ✅ | معرفات المشرفين مفصولة بفاصلة، مثال: `123456789,987654321` |
| `MINIAPP_URL` | ✅ | رابط HTTPS للمنصة (Mini App) |
| `REQUIRED_CHANNEL_ID` | اختياري | معرف القناة لبوابة العضوية، مثال: `-1001234567890` |
| `REQUIRED_CHANNEL_USERNAME` | اختياري | اسم مستخدم القناة بدون @، مثال: `mychannel` |

### المنصة (Mini App — Next.js)
| المتغير | مطلوب | الوصف |
|---------|:---:|-------|
| `BOT_TOKEN` | ✅ | نفس توكن البوت (للتحقق من initData) |
| `ADMIN_IDS` | ✅ | نفس المشرفين |
| `OPENROUTER_API_KEY` | ✅ | مفتاح OpenRouter للذكاء الاصطناعي — احصل عليه من [openrouter.ai](https://openrouter.ai) |
| `OPENROUTER_MODEL` | اختياري | نموذج الاستخراج (الافتراضي: `google/gemini-2.5-flash-preview`) |
| `OPENROUTER_GRADING_MODEL` | اختياري | نموذج التصحيح (الافتراضي: نفس `OPENROUTER_MODEL`) |
| `OPENROUTER_FALLBACK_MODELS` | اختياري | نماذج احتياطية مفصولة بفاصلة |
| `MINIAPP_URL` | اختياري | رابط المنصة (لإشعارات الامتحان الجديد) |
| `REQUIRED_CHANNEL_ID` | اختياري | نفس معرف القناة |
| `REQUIRED_CHANNEL_USERNAME` | اختياري | نفس اسم القناة |
| `BOT_DB_PATH` | اختياري | مسار قاعدة البيانات (الافتراضي: `../bot.db`) |
| `UPLOAD_DIR` | اختياري | مجلد حفظ صور الأوراق (الافتراضي: `./uploads`) |
| `TZ_OFFSET_MS` | اختياري | فارق التوقيت بالميلي ثانية للتصنيف (الافتراضي: `10800000` = UTC+3) |
| `SCORE_SPEED_TIERS` | اختياري | طبقات مكافأة السرعة (الافتراضي: `1:15,3:10,6:5,24:2`) |
| `SCORE_LOYALTY_STEP` | اختياري | خطوة مكافأة الوفاء بالنسبة المئوية (الافتراضي: `3`) |
| `SCORE_LOYALTY_CAP` | اختياري | الحد الأقصى لمكافأة الوفاء (الافتراضي: `15`) |
| `COMP_STAKES` | اختياري | رهانات المسابقات مفصولة بفاصلة (الافتراضي: `10,25,50,100`) |

## أوامر البوت

| الأمر | الوصف |
|-------|-------|
| `/start` | رسالة الترحيب + زر فتح التطبيق |
| `/app` أو `/open` | فتح التطبيق مباشرةً |
| `/exams` | الانتقال لقائمة الامتحانات المتاحة |
| `/results` | الانتقال لنتائجك وتصحيحاتك |
| `/history` | سجل إجاباتك السابقة |
| `/students` | (للمشرف) قائمة الطلاب ودرجاتهم |
| `/new` | (للمشرف) تسجيل امتحان جديد |

## البنية التقنية

```
المشروع/
├── index.ts              # بوت تيليجرام (Telegraf) — launcher + outbox drainer
├── src/
│   ├── config.ts         # تهيئة البوت من env vars
│   ├── db.ts             # قاعدة بيانات البوت (bun:sqlite) — users + outbox فقط
│   └── membership.ts     # بوابة العضوية في القناة
└── miniapp/              # المنصة (Next.js)
    ├── app/
    │   ├── page.tsx      # Shell: تحميل المستخدم + التوجيه بين الصفحات
    │   ├── api/          # مسارات API (auth, exams, submit, leaderboard, ...)
    │   └── components/   # واجهات الطالب والمشرف
    └── lib/
        ├── db.ts         # طبقة البيانات الكاملة (better-sqlite3)
        ├── gemini.ts     # عميل OpenRouter للذكاء الاصطناعي
        ├── auth.ts       # التحقق من initData تيليجرام
        ├── scoring.ts    # مكافآت السرعة والوفاء
        └── uploads.ts    # حفظ الصور على القرص
```

## تشغيل المشروع

```bash
# تشغيل البوت
bun run index.ts

# تشغيل المنصة (development)
cd miniapp && pnpm dev

# تشغيل المنصة (production)
cd miniapp && pnpm build && pnpm start
```

## كيفية العمل

1. **الأستاذ** يفتح التطبيق → **تسجيل امتحان** → يرفع صور مفتاح الإجابة
2. الذكاء الاصطناعي يستخرج الأسئلة والإجابات النموذجية تلقائياً
3. الأستاذ يراجع المفتاح ويؤكد التفعيل
4. يصل إشعار للطلاب بالبوت بوجود امتحان جديد
5. **الطالب** يفتح التطبيق → **الامتحانات** → يختار الامتحان → يرفع صور ورقته
6. التصحيح يظهر فورياً مع الدرجة والملاحظات التفصيلية بالعربية
7. تُضاف النقاط للرصيد مع مكافآت السرعة والوفاء
